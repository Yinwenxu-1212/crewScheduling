# CTS 2025 机组排班问题求解器 (Crew Scheduling Solver)

## 1. 项目概述

本项目是为 "CTS 2025 算法大赛" 设计的机组排班问题求解器。项目采用运筹优化领域中处理大规模问题的经典算法——**列生成 (Column Generation)**——作为核心框架，并使用 Python 语言进行实现，优化求解器为 Gurobi。

该求解器旨在为所有机组人员设计一套成本最低、满足所有复杂约束、并能在竞赛评分标准下获得高分的周度排班计划。

## 2. 核心功能与已实现规则

经过我们的逐步开发，当前版本的求解器已经具备了处理复杂航空排班规则的能力：

- **核心优化框架**:
    - [x] 基于**列生成 (Column Generation)** 的完整算法流程。
    - [x] 主问题采用**集合覆盖模型**，并引入**人工变量**确保初始解的可行性。
- **已实现的硬性约束**:
    - [x] **机组资质**: (隐式实现) 子问题为每个机组单独生成排班。
    - [x] **特殊置位规则**: 严格遵守“置位只能发生在值勤日的开始或结束”的竞赛规则。
    - [x] **占位任务约束**: 能够自动规避与机组预设的培训、休假等任务的时间冲突。
    - [x] **法定休息期**: 在两个值勤日之间强制插入不低于12小时的休息期。
    - [x] **累计飞行时间限制**: 遵守任意连续168小时（7天）内，累计飞行不超过40小时的规则。
    - [x] **最长值勤日限制**: 限制单次连续值勤不超过14小时。
- **与评分标准对齐的优化**:
    - [x] **成本函数设计**: 子问题的成本函数与官网评分标准挂钩，综合考虑了飞行小时奖励、外站过夜惩罚、置位次数惩罚、占用日历天惩罚等。
    - [x] **硬约束规避**: 杜绝“违规次数”和“新增过夜站点”的扣分，将这些视为不可行的路径。
- **性能与竞赛要求**:
    - [x] **2小时运行限制**: 内置“熔断”机制，确保程序能在2小时内运行完毕并输出结果。

## 3. 项目文件结构

本项目采用模块化设计，各文件职责清晰：

```
/crew_scheduling_project/
|
|-- data/                     # 存放所有输入数据 .csv 文件
|   |-- flight.csv
|   |-- crew.csv
|   |-- groundDuty.csv
|   |-- bus.csv
|   |-- layoverStation.csv
|   |-- crewLegMatch.csv
|
|-- main.py                   # 程序的总入口，负责组织列生成的完整流程
|-- data_models.py            # 定义所有核心数据结构 (如 Flight, Crew, Roster)
|-- data_loader.py            # 从 CSV 文件中读取原始数据并转化为对象
|-- master_problem.py         # 主问题模块：负责“选择”最优的排班组合
|-- subproblem_solver.py      # 子问题模块：负责“生成”新的、合规的排班方案（核心逻辑所在地）
|-- results_writer.py         # 结果输出模块：生成符合竞赛要求的提交文件
|-- README.md                 # (本文档) 项目说明文件
```

## 4. 环境配置与安装

在运行本项目前，请确保您的环境中已安装 Python (建议 3.8 或更高版本) 和以下必要的库。

1.  **安装 Gurobi**:
    请根据 Gurobi 官网的指引安装 Gurobi Optimizer，并确保您已正确配置许可证（例如，免费的学术许可证）。

2.  **安装依赖库**:
    建议使用 `pip` 安装其他必要的 Python 库。您可以创建一个 `requirements.txt` 文件，内容如下：
    ```txt
    # requirements.txt
    pandas
    gurobipy
    ```
    然后通过以下命令安装：
    ```bash
    pip install -r requirements.txt
    ```

## 5. 如何运行

1.  **准备数据**:
    确保所有竞赛提供的 `flight.csv`, `crew.csv` 等数据文件都存放在项目根目录下的 `data` 文件夹中。

2.  **执行程序**:
    在项目的根目录下，打开终端或命令行，运行 `main.py` 文件：
    ```bash
    python main.py
    ```
    程序将开始运行，并在终端中打印出每一轮迭代的信息。由于内置了时间限制，程序最长将在约1小时55分钟后自动结束。

## 6. 输出结果

程序运行结束后，将在项目根目录下生成一个名为 `rosterResult.csv` 的文件。该文件严格按照竞赛要求的格式生成，包含三列：
* `crewId`: 机组人员ID。
* `taskId`: 任务的唯一ID（来源于原始数据）。
* `isDDH`: 是否为置位任务（1表示是，0表示否）。
